## How to Contribute

### 1. Fork & Clone
```bash
git clone https://github.com/YOUR_USERNAME/ai-sensei.git
cd ai-sensei
```

### 2. Create a Feature Branch
```bash
git checkout -b feature/your-feature-name
```

### 3. Make Your Changes
- Add features, fix bugs, improve documentation
- Keep code style consistent with existing codebase
- Add comments for complex logic

### 4. Test Your Changes
- Run the notebook cells to verify functionality
- Test with sample data before submitting PR

### 5. Commit & Push
```bash
git add .
git commit -m "feat: description of your change"
git push origin feature/your-feature-name
```

### 6. Submit a Pull Request
- Describe what your PR does
- Reference any related issues
- Include example usage if applicable

## Code Style

- Use Python 3.8+ syntax
- Follow PEP 8 naming conventions
- Add docstrings to functions
- Include type hints where possible

## Reporting Issues

Found a bug? Please create an issue with:
- Description of the problem
- Steps to reproduce
- Expected vs. actual behavior
- Your environment (OS, Python version, etc.)

Feel free to open a discussion or reach out!
